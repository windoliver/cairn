# Docs

The docs site source lives in `docs/site/` and is built by mdBook.

Local workflow:

```bash
cargo run -p cairn-cli --bin cairn-docgen --locked -- --write
mdbook build docs/site
```

Docs generated by `cairn-docgen` are committed. mdBook HTML output is not
committed.

When adding a user-facing surface:

- New CLI command or flag: update the runtime clap tree, run `cairn-docgen`.
- New config field/default: update config types, run `cairn-docgen`.
- New bundled plugin: update the registry and `docs/site/docs-coverage.toml`,
  run `cairn-docgen`.
- New workspace package: add a package entry to `docs/site/docs-coverage.toml`.
- New IDL verb or metadata: run `cairn-codegen`, then `cairn-docgen`.
