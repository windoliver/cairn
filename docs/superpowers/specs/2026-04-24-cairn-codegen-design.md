## 1. Problem

The IDL under `crates/cairn-idl/schema/` is the canonical contract for Cairn's eight verbs and two protocol preludes (`status`, `handshake`). Today the binary `cairn-codegen` is a stub that exits non-zero. Four downstream surfaces ÔÇö Rust SDK types, the `cairn` CLI's clap definitions, MCP tool declarations, and the shippable Cairn skill (SKILL.md) ÔÇö must all derive from that one IDL so they cannot drift.

This spec describes the codegen pipeline that produces those four artefact bundles deterministically, the layout the generated files take, and the CI gate that proves the committed outputs are in sync with the IDL.

## 2. Goals & non-goals

### Goals

1. Read the IDL and emit four deterministic artefact bundles: SDK types, CLI clap commands, MCP tool declarations + JSON schemas, Cairn skill bundle.
2. Generated outputs are committed to the repository; CI validates no drift.
3. A second run of the generator on a clean tree is a no-op (idempotent).
4. All four surfaces enumerate the **same eight verb ids** in the same order; `status` and `handshake` are flagged as protocol preludes and excluded from the eight-verb count everywhere.
5. Maintainer docs name the single command to run when the IDL changes.

### Non-goals

- Implementing verb behaviour (persistence, search ranking, MCP transport runtime). Out of scope per the issue.
- A TypeScript SDK. The brief (┬ž13.5) puts a TS SDK at P1+; no TS workspace exists today. The cross-language need is satisfied by publishing the JSON Schemas as a runtime artefact (any TS / Python client reads them at MCP `initialize`).
- Compile-time validation that generated Rust type-checks. `cargo check --workspace` already covers this; codegen does not re-implement type-checking.
- Runtime depth validation on the `Filter` enum. JSON Schema enforces depth ÔëĄ 8 at ingest; the Rust enum is unbounded. The runtime depth check lives in the search verb implementation (#9 / #63).
- Broader contract-drift gates beyond no-diff (covered by #36); JSON-Schema meta-validation (covered by #98); skill-install validation (covered by #70).

## 3. Architecture

`cairn-codegen` is a maintainer-time binary that reads the IDL under `crates/cairn-idl/schema/` and writes deterministic source files into four locations. Run manually (`cargo run -p cairn-idl --bin cairn-codegen`); CI re-runs it in `--check` mode and asserts no diff.

### 3.1 Generator structure (`crates/cairn-idl/`)

```
src/
  lib.rs                          ÔćÉ keeps SCHEMA_DIR; re-exports codegen::run
  codegen/
    mod.rs                        ÔćÉ public API: run(opts) -> Result<Report>
    loader.rs                     ÔćÉ parse IDL files into Document (typed)
    ir.rs                         ÔćÉ internal IR: Verb, Schema, Field, VariantÔÇŽ
    emit_sdk.rs                   ÔćÉ Rust types Ôćĺ cairn-core/src/generated/
    emit_cli.rs                   ÔćÉ clap Commands Ôćĺ cairn-cli/src/generated/
    emit_mcp.rs                   ÔćÉ tool decls + JSON schemas Ôćĺ cairn-mcp/src/generated/
    emit_skill.rs                 ÔćÉ SKILL.md + conventions.md + .version Ôćĺ skills/cairn/
    fmt.rs                        ÔćÉ deterministic JSON serialiser, sort helpers
  bin/cairn-codegen.rs            ÔćÉ thin CLI: parse args, call codegen::run
```

### 3.2 Generated outputs

Every generated file carries `// @generated by cairn-codegen ÔÇö DO NOT EDIT.` (or `<!-- @generated -->` for markdown) at the top so reviewers spot manual edits.

```
crates/cairn-core/src/generated/
  mod.rs                ÔćÉ pub mod verbs; pub mod common; pub mod errors;
  verbs/
    mod.rs              ÔćÉ VerbId enum, VERBS registry, capability lists
    ingest.rs           ÔćÉ IngestArgs, IngestData
    search.rs           ÔćÉ SearchArgs, SearchData, SearchHit, SearchMode, Filter (recursive), Trust
    retrieve.rs         ÔćÉ RetrieveArgs (tagged enum), per-variant Data structs
    summarize.rs        ÔÇŽ (one file per verb)
    ÔÇŽ
  common/
    mod.rs              ÔćÉ Ulid, Cursor, ScopeFilter, SignedIntent, Envelope
  errors/
    mod.rs              ÔćÉ CairnError enum + ErrorCode enum
  status.rs             ÔćÉ StatusResponse type (prelude ÔÇö flagged separately from VERBS)
  handshake.rs          ÔćÉ ChallengeResponse type

crates/cairn-cli/src/generated/
  mod.rs                ÔćÉ pub fn command() -> clap::Command (builds full subcommand tree)
  verbs.rs              ÔćÉ per-verb subcommand builders + parse fns
  prelude.rs            ÔćÉ status / handshake subcommands

crates/cairn-mcp/src/generated/
  mod.rs                ÔćÉ pub const TOOLS: &[ToolDecl]; ToolDecl { name, description, input_schema }
  schemas/              ÔćÉ canonical JSON-Schema bytes (cross-language artefact)
    verbs/<verb>.json
    prelude/<status|handshake>.json
    common/*.json

skills/cairn/
  SKILL.md              ÔćÉ markdown contract built from x-cairn-skill-triggers blocks
  conventions.md        ÔćÉ kind cheat-sheet (regenerated against VerbId; kinds detail TBD until #4)
  .version              ÔćÉ contract id (cairn.mcp.v1) + cairn-idl pkg version
```

**Rationale.** Co-located output (per consumer crate) keeps each crate self-contained, plays nice with rust-analyzer, and matches the brief's `src/<surface>/` layout. The CI gate is identical to a centralised layout. SDK files embed schema bytes via `include_bytes!` reaching across to `cairn-mcp/src/generated/schemas/` so there is exactly one byte-source for each schema; the embed sites are exercised by `codegen_surface_parity` so a workspace move would be caught at compile time.

## 4. Generator pipeline

Three pure stages: **load Ôćĺ IR Ôćĺ emit**. Each stage is testable with in-memory schema fixtures.

### 4.1 Load (`codegen/loader.rs`)

Walks `index.json`, opens every file under `x-cairn-files`, deserialises into `serde_json::Value`, and validates structural invariants (re-using the assertions already in `tests/schema_files.rs`):

- `x-cairn-contract == "cairn.mcp.v1"` on every file.
- Every verb file has `x-cairn-verb-id`, `x-cairn-cli`, `x-cairn-skill-triggers`, `$defs/Args`, `$defs/Data`.
- Every `$ref` resolves to a real path within `SCHEMA_DIR`.
- Every `x-cairn-capability` value appears in `capabilities/capabilities.json`.

Any failure here is a hard error returned from `codegen::run`; the binary exits non-zero **before** writing anything, so half-written trees are not possible.

### 4.2 Lower to IR (`codegen/ir.rs`)

A typed Rust IR insulates emitters from raw JSON.

```rust
pub struct Document {
    pub contract: ContractId,            // "cairn.mcp.v1"
    pub capabilities: Vec<Capability>,   // closed enum from capabilities.json
    pub error_codes: Vec<ErrorCode>,     // closed enum from errors/error.json
    pub common: BTreeMap<TypeName, RustType>,
    pub verbs: Vec<VerbDef>,             // ordered per index.json (NOT alphabetised)
    pub preludes: Vec<PreludeDef>,       // status, handshake
}

pub struct VerbDef {
    pub id: VerbId,
    pub args: RustType,
    pub data: RustType,
    pub cli: CliShape,                   // x-cairn-cli block
    pub skill: SkillBlock,               // x-cairn-skill-triggers
    pub capability: Option<Capability>,
    pub auth: AuthModel,                 // signed_chain | rebac | hardware_key | ÔÇŽ
    pub args_schema_bytes: Vec<u8>,      // canonical JSON for embedding
    pub data_schema_bytes: Vec<u8>,
}

pub enum RustType {
    Primitive(Prim),                     // String, i64, u64, f64, bool
    Optional(Box<RustType>),
    Vec(Box<RustType>),
    Map(Box<RustType>),                  // HashMap<String, T>
    Ref(TypeName),                       // resolved $ref
    Struct(StructDef),                   // additionalProperties: false
    Enum(EnumDef),                       // string enum or oneOf-of-const
    TaggedUnion(TaggedUnionDef),         // RetrieveArgs (oneOf with discriminator field)
    UntaggedUnion(UntaggedUnionDef),     // ingest Args' body|file|url XOR
    Recursive(RecursiveEnumDef),         // Filter (collapses filter_L0..L8)
    Json,                                // serde_json::Value (frontmatter, opaque blobs)
}
```

**Lowering rules.**

| Schema construct | IR / Rust output |
|---|---|
| `"type": "object"` + `additionalProperties: false` + `required: [...]` | `Struct` with non-Option for required fields, `Option<T>` for the rest, `#[serde(deny_unknown_fields)]` |
| `"type": "object"` + `additionalProperties: true` (frontmatter) | `serde_json::Map<String, Value>` field |
| `"type": "string"` + `enum: [...]` | `Enum` with `#[serde(rename_all = "snake_case")]` |
| `"oneOf": [{const: "x"}, {const: "y"}]` (e.g. `search.mode`) | `Enum` (each `const` becomes a variant) |
| `"oneOf": [...]` with explicit `x-cairn-discriminator: "<field>"` | `TaggedUnion` Ôćĺ `#[serde(tag = "<field>", rename_all = "snake_case")]` |
| `"oneOf": [{required:[A]}, {required:[B]}, ÔÇŽ]` (e.g. ingest XOR) | All fields stay as `Option<T>`; emit a `validate()` constructor returning `Result<Args, ValidationError>` enforcing exactly-one |
| `$ref: "../common/primitives.json#/$defs/Ulid"` | `Ref(TypeName::new("Ulid"))` |
| `filter` family (`filter_leaf` + `filter_L0..L8`) | Detected by name pattern Ôćĺ collapse to one `pub enum Filter { Leaf(FilterLeaf), And(Vec<Filter>), Or(Vec<Filter>), Not(Box<Filter>) }`. Depth bound stays in JSON Schema only (Rust runtime check deferred to #9 / #63). |

### 4.3 IDL extension required: `x-cairn-discriminator`

`RetrieveArgs` (┬ž8.0.c) is a tagged union on the `target` field. The current IDL expresses this as a `oneOf` of variants without a discriminator marker, which is ambiguous from the lowering side. This PR adds a single new vendor extension:

```jsonc
"oneOf": [
  { "$ref": "#/$defs/RetrieveArgs.Record" },
  { "$ref": "#/$defs/RetrieveArgs.Session" },
  ÔÇŽ
],
"x-cairn-discriminator": "target"
```

Purely additive ÔÇö does not break existing schema-files tests. A new test (`tests/schema_discriminator.rs`) asserts every `oneOf` either carries `x-cairn-discriminator` or is in a known-no-discriminator allow-list (string-const enums; XOR-required patterns).

**Fallback if the IDL stays frozen.** Hand-code a small allow-list inside the loader keyed on `(verb_id, $defs path)` Ôćĺ discriminator field name. Less clean but keeps the IDL untouched. Decision recorded with the PR.

### 4.4 Emit

Each emitter takes `&Document` and returns `Vec<GeneratedFile>` (path + bytes). Emitters never touch the filesystem directly ÔÇö `codegen::run` writes the union of all emitter outputs in a single batched pass after every emitter has succeeded. **Partial failure is impossible**: either all files land or none do.

**`emit_sdk.rs`** (Ôćĺ `crates/cairn-core/src/generated/`):
- One file per verb, plus shared `common/`, `errors/`, `verbs/mod.rs` (the registry).
- Tiny string-builder (no `quote!` / `syn` ÔÇö keeps deps zero, output stable).
- `#![allow(clippy::large_enum_variant, clippy::module_name_repetitions, clippy::missing_errors_doc)]` at module top ÔÇö only allow lints, justified inline.
- Schemas embedded via `pub const ARGS_SCHEMA: &[u8] = include_bytes!("../../../../cairn-mcp/src/generated/schemas/verbs/ingest.json");` ÔÇö single byte source, no duplication.

**`emit_cli.rs`** (Ôćĺ `crates/cairn-cli/src/generated/`):
- Builds `clap::Command` via the **builder API** (not derive). Builder is fully programmatic, so emitted code is mechanical and diff-stable.
- One `pub fn command() -> clap::Command` returning the full subcommand tree (8 verbs + `status` + `handshake`).
- Per-verb `pub fn parse_<verb>(matches: &ArgMatches) -> Result<<Verb>Args, CliError>` so cairn-cli's `main.rs` only does dispatch.

**`emit_mcp.rs`** (Ôćĺ `crates/cairn-mcp/src/generated/`):
- `mod.rs` exports `pub const TOOLS: &[ToolDecl]` array (8 verbs, in IDL order).
- `ToolDecl { name, description, input_schema, capability, auth }` ÔÇö `description` built from `x-cairn-skill-triggers` (one-line purpose + positive triggers + negative triggers + exclusivity hint), exactly matching brief ┬ž8.0.b's "MCP tool descriptions are prompt-engineered" requirement.
- `schemas/` subtree holds the JSON schemas as plain files, one per verb / prelude / common ref. **Cross-language artefact** for any non-Rust client.

**`emit_skill.rs`** (Ôćĺ `skills/cairn/`):
- `SKILL.md` built from ┬ž18.d template + `x-cairn-skill-triggers` blocks for each verb.
- `conventions.md` is the kind cheat-sheet, regenerated against `VerbId`. Kinds detail integrates later when the taxonomy IDL slice (#4) lands; until then conventions.md carries a marker comment naming that issue.
- `.version` carries the contract id + the `cairn-idl` workspace package version (read at codegen time).

## 5. Determinism, idempotency, CI gate

### 5.1 Determinism contract

Three rules every emitter obeys (enforced by tests, not convention):

1. **Stable iteration.** Verbs in `index.json` order; type members in `BTreeMap` order; capabilities sorted lexicographically. No `HashMap`, no `HashSet`, no `rayon::par_iter`, no `SystemTime::now()`, no `Uuid::new_v4()` ÔÇö codegen has zero ambient sources of non-determinism.
2. **Canonical serialisation.** JSON written via a single helper `fmt::write_json_canonical(&Value)` that sorts keys recursively, two-space indent, trailing newline. Rust written via emitters that always end files with exactly one trailing `\n`.
3. **Atomic writes.** Each output file goes through `tempfile::NamedTempFile::persist` so a panic mid-write cannot leave a half-formed source file.

### 5.2 `--check` mode

```
cairn-codegen                  # write outputs to canonical workspace paths (default)
cairn-codegen --check          # generate to in-memory buffers; diff against on-disk; exit 1 on drift
cairn-codegen --out <DIR>      # write to a custom root (used by tests; default = workspace root resolved from CARGO_MANIFEST_DIR/..)
```

`--check` does **not** invoke `git`; it compares the generator's in-memory output bytes against the on-disk file bytes directly. Independent of git config, line-ending settings, and uncommitted index state.

### 5.3 CI gate

Add one job to `.github/workflows/ci.yml`:

```yaml
codegen-drift:
  name: codegen / no drift
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - uses: dtolnay/rust-toolchain@1.95.0
    - run: cargo run -p cairn-idl --bin cairn-codegen -- --check
```

Failure prints first 20 differing files with relative paths. Fix is "run `cargo run -p cairn-idl --bin cairn-codegen` and commit the diff". Issue #36 is the broader contract-drift epic; this job is the slice that lands with #35.

### 5.4 Tests (`crates/cairn-idl/tests/`)

| Test | Asserts |
|---|---|
| `codegen_loader.rs` | Loader rejects malformed IDL: missing `x-cairn-verb-id`, dangling `$ref`, unknown capability, missing `$defs/Args`. One test per failure mode. |
| `codegen_idempotent.rs` | Run codegen Ôćĺ snapshot output bytes Ôćĺ run codegen again Ôćĺ assert byte-equal. The "no-op on second run" verification. |
| `codegen_determinism.rs` | Run codegen 5 times in a fresh tempdir each Ôćĺ all five outputs byte-equal. Catches accidental hash-iteration leaks. |
| `codegen_surface_parity.rs` | Acceptance criterion: confirms `VerbId` enum (SDK), `command()` subcommands (CLI), `TOOLS` array (MCP), and `SKILL.md` headings all enumerate the same eight verb ids in the same order. Excludes `status` and `handshake`. |
| `codegen_snapshot.rs` | `insta` snapshots of representative emitter outputs (one verb's SDK file, the CLI `command()` skeleton, MCP `ingest.json`, SKILL.md). Catches accidental output drift in PR review even when CI is green. |
| `codegen_check_mode.rs` | `--check` returns 0 on a clean tree, returns 1 with stable error message after touching one generated file. |

Runs under existing `cargo nextest run --workspace`.

### 5.5 What this guarantees vs. what it does not

**Guarantees**:
- Clean checkout + `cargo run -p cairn-idl --bin cairn-codegen` Ôćĺ zero diff.
- Second run is a no-op.
- All four surfaces enumerate the same eight verbs.
- `status` / `handshake` are flagged separately and excluded from the eight-verb count everywhere.

**Does not guarantee** (out of scope):
- Generated Rust compiles. `cargo check --workspace` covers this.
- MCP JSON schemas validate against draft-2020-12. Adding meta-validation is #98.
- Generated `Filter` enum's recursive depth ÔëĄ 8 at runtime. Documented as a deliberate trade-off; the runtime check lives in the search verb (#9 / #63).

## 9. Risks called out for review

- **Touching the closed IDL.** Adding `x-cairn-discriminator` is the only IDL change. Purely a vendor extension on the `oneOf` for `RetrieveArgs`; no consumer of the existing schema breaks. Fallback if the IDL stays frozen: hand-code a small allow-list inside the loader keyed on `(verb_id, $defs path)` Ôćĺ discriminator field name.
- **Embedded `include_bytes!` paths.** SDK files embed schema bytes via relative paths reaching across crate boundaries. If the workspace is restructured these break. Mitigation: each `include_bytes!` site has a comment + `codegen_surface_parity` exercises the include at compile time.
- **`Filter` recursion bound.** Rust enum is unbounded; JSON Schema bound stays at depth 8. Hand-crafted deeply nested values bypass the schema; depth check lives in the search verb (#9 / #63), not here. Documented in `docs/dev/codegen.md`.

## 10. Invariants this PR touches (CLAUDE.md ┬ž4)

- **Invariant 3 ÔÇö CLI is ground truth.** This PR makes that literal: every surface's verb list comes from the same IDL.
- **Invariant 4 ÔÇö Seven contracts, pure functions otherwise.** Codegen is a pure function (load Ôćĺ IR Ôćĺ emit); no hidden state.
- **Invariant 6 ÔÇö Fail closed on capability.** Generated tool decls carry `capability` per verb; runtime gating ships in #9 / #64.

No invariants weakened.
